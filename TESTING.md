# Testing Lox++ Parser, Compiler, and VM

This document describes how to write and run tests for the Lox++ parser, bytecode compiler, and VM. It covers:
- How to add new expression evaluator and bytecode generator tests
- Example test cases
- How to interpret test failures
- Test file locations and helpers
- Setup and build requirements

## Test Infrastructure Overview

Lox++ uses [GoogleTest (GTest)](https://github.com/google/googletest) for C++ unit testing. The test harness provides helpers to:
- Evaluate Lox++ expressions and check their runtime result (VM tests)
- Compile expressions and check the generated bytecode (bytecode tests)

## Writing Expression Evaluator (VM) Tests

Add new tests to `test/test_parser_vm.cpp`. Use helpers from `test/test_harness.h`:
- `expect_number_eq(expr, expected)` — checks numeric result
- `expect_bool_eq(expr, expected)` — checks boolean result
- `expect_string_eq(expr, expected)` — checks string result
- `expect_nil(expr)` — checks for nil

**Example:**
```cpp
TEST_F(ParserVMTest, Arithmetic) {
    expect_number_eq("1 + 2 * 3", 7);
    expect_number_eq("(1 + 2) * 3", 9);
}

TEST_F(ParserVMTest, BooleanLogic) {
    expect_bool_eq("true and false", false);
    expect_bool_eq("!true", false);
}

TEST_F(ParserVMTest, StringConcat) {
    expect_string_eq("\"foo\" + \"bar\"", "foobar");
}
```

## Writing Bytecode Generator Tests

Add new tests to `test/test_parser_bytecode.cpp`. Use the `compile_to_bytecode(expr)` helper to get a string representation of the bytecode. Compare it to the expected output.

**Example:**
```cpp
TEST_F(ParserBytecodeTest, Arithmetic_Addition) {
    std::string expr = "1 + 2";
    std::string bytecode = compile_to_bytecode(expr);
    std::string expected =
        "0: CONSTANT 0 ('1')\n"
        "2: CONSTANT 1 ('2')\n"
        "4: ADD\n"
        "5: RETURN\n";
    EXPECT_EQ(trim(bytecode), trim(expected));
}
```

## Interpreting Test Failures

- **VM tests:** A failure means the evaluated result did not match the expected value. The error message will show the actual and expected values.
- **Bytecode tests:** A failure means the generated bytecode does not match the expected string. The diff will show where the output diverges.

## Test Files and Helpers

- Test sources: `test/test_parser_vm.cpp`, `test/test_parser_bytecode.cpp`
- Test harness: `test/test_harness.h`, `test/test_harness.cpp`
- Build/test config: `test/CMakeLists.txt`

## Setup and Build Requirements

- Requires CMake and GTest (see README.md for dependencies)
- To build and run all tests:
  ```bash
  cmake -B build -G Ninja && cmake --build build
  ctest --test-dir build
  ```
- To run a specific test binary (e.g., for debugging):
  ```bash
  ./build/test_main
  ```

## Adding New Tests

1. Add your test case to the appropriate file.
2. Use the helpers for concise assertions.
3. Rebuild and run tests to verify.

For more details, see the comments in the test harness and example test files.
